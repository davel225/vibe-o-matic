# vibe-o-matic x402 endpoint

`POST /api/vibeify/x402` is the **machine-callable** entry point. External agents pay $0.69 USDC (Base mainnet) per render via the [x402 protocol](https://x402.org) and receive a Vibetown vinyl-figurine render of the supplied photo.

The endpoint runs the **same** describe → prompt → Flux pipeline as the web UI. Two calling modes are supported:

| Mode | When to use | What the caller sends |
|---|---|---|
| **Explicit** | Caller already knows what scene/action/mood/size they want | `image` + `scene` + `action` + `mood` + `size` + `sceneBgImages` |
| **Agent** | Caller just has a photo + free-text intent; server picks the rest | `image` + `agentMode=1` + (optional) `intent` |

## Discovery

```http
GET /api/vibeify/x402
```

Returns the x402 facilitator URL, network, price, and recipient address. No payment required.

## Quickest way to test (CLI, ~30s)

The repo ships a self-contained agent runner that handles discovery, EIP-3009 signing, balance preflight, and result-saving. Clone, install, run:

```bash
git clone https://github.com/davel225/vibe-o-matic
cd vibe-o-matic
npm install

# Default: looks for ./agent-photo.{jpg,png,jpeg,webp} in cwd
AGENT_PRIVATE_KEY=0x... node scripts/test-x402-agent.mjs "your free-text intent"
```

The runner accepts flags so you can wire it into any credential-store convention:

```bash
node scripts/test-x402-agent.mjs --help

# Common variants:
node scripts/test-x402-agent.mjs -i ./my-photo.jpg "rockstars at sunset"
node scripts/test-x402-agent.mjs -k ~/.config/vibe-o-matic/key "chill day"

# Source the key from any JSON credential store:
AGENT_PRIVATE_KEY=$(jq -r .privateKey ~/.your-agent/creds.json) \
  node scripts/test-x402-agent.mjs "your intent"
```

What it does:
- Auto-normalizes the private key (0x prefix optional)
- Defaults the image to `./agent-photo.{jpg,jpeg,png,webp}` if `--image` isn't supplied
- Pre-flights your USDC balance on Base mainnet — fails fast if you can't cover the $0.69 charge (skip with `--no-balance-check`)
- Runs the full discover → sign → retry handshake transparently via `x402-fetch`
- Prints the agent's scene/action/mood/size picks + reasoning
- Saves the rendered PNG to `./x402-agent-out-<timestamp>.png`
- Prints the settlement tx hash + Basescan link

## Calling from your own code

### x402 payment flow (both modes)

1. `POST` once without an `X-PAYMENT` header → server returns `402 Payment Required` with `accepts` (EIP-3009 payment requirements).
2. Sign the EIP-3009 authorization with your wallet, encode as the `X-PAYMENT` header.
3. `POST` again with the header → server verifies, runs the render, settles the USDC transfer atomically with the response, and returns the image.

In TypeScript, use [`x402-fetch`](https://www.npmjs.com/package/x402-fetch) which handles the two-step flow transparently:

```ts
import { wrapFetchWithPayment } from "x402-fetch";
import { createWalletClient, http } from "viem";
import { base } from "viem/chains";
import { privateKeyToAccount } from "viem/accounts";

const account = privateKeyToAccount(process.env.AGENT_PRIVATE_KEY as `0x${string}`);
const wallet = createWalletClient({ account, chain: base, transport: http() });
const fetchWithPay = wrapFetchWithPayment(fetch, wallet);
```

### Explicit mode

```ts
const form = new FormData();
form.set("image", imageBlob);          // or: form.set("imageUrl", "https://...")
form.set("scene", "a sun-drenched tropical beach with palm trees...");
form.set("action", "leaning in for a group selfie...");
form.set("mood", "warm joyful glow — golden-hour light...");
form.set("size", "1536x1024");          // 1024x1024 | 1024x1536 | 1536x1024
form.set("sceneBgImages", "tropical-beach.webp");  // comma-separated

const res = await fetchWithPay("https://your-host/api/vibeify/x402", {
  method: "POST",
  body: form,
});
const { image, prompt, description } = await res.json();
// image is a data: URL
```

### Agent mode (recommended for autonomous agents)

The caller doesn't need to know any of vibe-o-matic's preset ids or prompt structure. They send the photo and (optionally) a one-line intent; our server picks the scene/action/mood/size from the curated catalog using gpt-4o-mini vision.

```ts
const form = new FormData();
form.set("image", imageBlob);
form.set("agentMode", "1");
form.set("intent", "make us look like rockstars at a wild after-party");  // optional

const res = await fetchWithPay("https://your-host/api/vibeify/x402", {
  method: "POST",
  body: form,
});
const { image, prompt, description, agentPicks } = await res.json();
// agentPicks = {
//   sceneId: "neon-street",
//   actionId: "celebrate",
//   moodId: "hyped",
//   size: "1536x1024",
//   reasoning: "Two friends → group framing; neon scene matches 'wild after-party'"
// }
```

If intent is omitted, the agent decides purely from the photo's content (subject count, framing, apparent mood).

## Response shape

Both modes return the same JSON:

```ts
{
  image: string;          // data:image/png;base64,... — render is here
  prompt: string;         // full text prompt sent to Flux (for transparency)
  description: string;    // gpt-4o-mini's description of the source photo
  provider: "flux-2-pro";
  paymentRail: "usdc";
  agentMode: boolean;
  agentPicks?: {          // only present in agent mode
    sceneId: string;
    actionId: string;
    moodId: string;
    size: string;
    reasoning: string;
  };
}
```

Plus the `X-PAYMENT-RESPONSE` header containing the settled transaction details (base64-encoded JSON: `{ success, transaction, network, payer }`).

Source of truth: [`lib/presets.ts`](./lib/presets.ts).

## Error responses

| Status | Meaning |
|---|---|
| 400 | Bad form data / no image provided |
| 402 | Payment required, invalid, unverified, or settlement failed (body includes `accepts`) |
| 500 | Server misconfigured (`OPENAI_API_KEY` / `BFL_API_KEY` missing) |
| 502 | Upstream provider failed (describer, agent picker, or Flux) — body has `error` string |
